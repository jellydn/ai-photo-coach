## Codebase Patterns
- VisionCamera v5 uses Nitro modules architecture (react-native-nitro-modules)
- Camera component props changed: `outputs` array instead of `photo={true}`, no `testID` prop
- Jest tests require manual mocks for all native modules (vision-camera, permissions, safe-area-context)
- iOS permissions require NSCameraUsageDescription, NSPhotoLibraryUsageDescription, and NSMotionUsageDescription in Info.plist
- Android permissions need CAMERA, READ_EXTERNAL_STORAGE, WRITE_EXTERNAL_STORAGE, READ_MEDIA_IMAGES in AndroidManifest.xml
- Use @react-native-async-storage/async-storage for simple persistence (onboarding state, settings)
- Pattern: Create separate screen components for each onboarding step with shared navigation interface
- Pattern: Store onboarding completion state in AsyncStorage with key `@onboarding_complete`
- Pattern: Use View-based overlays with `pointerEvents="none"` for camera composition guides
- Pattern: Use percentage-based positioning (33.33% margins) for rule-of-thirds grid to scale across aspect ratios
- Pattern: React Native StyleSheet uses `absoluteFill` (not `absoluteFillObject`)
- Pattern: Create separate display metadata (UI strings/icons) from functional config (thresholds)
- Pattern: Sensor hooks should use RxJS Subscription type from 'rxjs', not from 'react-native-sensors'
- Pattern: react-native-sensors uses lowercase sensor types: `SensorTypes.accelerometer` (not `Accelerometer`)
- Pattern: Unsubscribe from sensors using `.unsubscribe()` method (RxJS), not `.remove()`
